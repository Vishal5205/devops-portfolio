About the Project

Project Overview

This project demonstrates a real-world DevOps CI/CD and GitOps workflow where application code is built, analyzed, containerized, and deployed to Kubernetes.

Due to SonarCloud free-tier limitations (scan delays and resource constraints), the code quality stage required manual validation. All remaining stages ‚Äî Jenkins build, Docker image creation, registry push, ArgoCD sync, and Kubernetes deployment ‚Äî are fully automated.

Deployment Process

Detailed Deployment Flow

Step 01
Developer pushes application source code to GitHub.
Step 02
GitHub triggers Jenkins pipeline. SonarCloud scan is manually validated.
Step 03
Jenkins checks out source code and builds Docker image.
Step 04
Docker image pushed to Docker Hub.
Step 05
ArgoCD syncs Kubernetes manifests.
Step 06
Kubernetes performs rolling update.

Automated Pipeline
Partial / Failed
When integrated into Jenkins CI/CD pipeline, SonarCloud scanning encounters issues and often fails to complete successfully or returns partial results.
üëâ Root Causes
  • SonarCloud Free Tier Limits: Restricted compute resources and analysis time for free accounts
  • Limited Compute Power: CI/CD pipeline runners have constrained CPU and memory
  • Missing Webhook Support: Free tier lacks webhook integration for automated PR analysis
  • API Rate Limiting: Frequent automated scans hit API rate limits quickly
  • Timeout Issues: Large codebases exceed free tier analysis time limits
